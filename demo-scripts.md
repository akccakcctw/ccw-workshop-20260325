# Demo 劇本 — 全球策略行銷處工作坊

## Phase 2：暖場 Demo

### Demo 1：「3 分鐘做一個網頁」（15 min）

**開場白**：
> 「在開始你們的工作內容之前，先讓你們看看 AI 的速度。
> 我們來即時做一個行銷活動頁面，請大家告訴我想要什麼主題？」

**Prompt（學員選完主題後，根據情境調整）**：

```
幫我做一個 KKday 櫻花季促銷 landing page，要有：
- 大標題 + 副標題
- 倒數計時器（到 4/15）
- 3 個精選商品卡片（含價格和折扣標籤）
- CTA 按鈕「立即搶購」
- 風格：春天、粉色系、日本櫻花
```

**互動點**：
- 生成後點開 Artifacts 預覽，**停頓 2-3 秒**讓學員看到完整網頁效果，再往下講
- 學員提修改：「換顏色」「加一個 banner」「改成英文版」
- 每次修改即時展示，強調「對話式來回修改」的威力
- 提醒學員：Artifacts 右上角可以全螢幕預覽、也可以直接複製程式碼

**講解重點**：
> 「注意我剛才做了什麼？我用自然語言描述我要什麼，AI 就自動幫我做出一個可預覽的網頁。
> 這跟你們平常跟工程師開需求的方式很像，但回應時間從幾天變成幾秒。」

### Demo 2：「讓 AI 讀懂你的資料」（15 min）

**開場白**：
> 「OK 寫網頁可能離你們有點遠，來看看跟你們工作更相關的。
> 假設你手上有一份客戶名單...」

**操作流程**：點左下角 📎 迴紋針 → 上傳 `data/customer_list.csv` → 輸入 Prompt

**Prompt**：

```
我上傳了一份客戶名單 CSV，幫我做以下分析：

1. 消費金額 Top 10 客戶是誰？列出他們的名字、國家、總消費和 VIP 等級
2. 各國家的客戶數和平均消費金額
3. 哪些客戶超過 3 個月沒消費了？（以今天 2026-03-25 計算）
4. 回購率最高的國家（用平均訂單數衡量）

最後幫我產一份可以直接貼到 email 的中文摘要。
```

**互動點**：
- 分析結果出來後，問學員：「你們平常做這件事要多久？」
- 讓學員追問：「那高消費但很久沒回來的客戶有哪些？」
- 展示 AI 能記住上下文，連續追問不用重新描述

**講解重點**：
> 「AI 不只看得懂資料，還能直接幫你寫結論、做建議。
> 以前你可能要開 Excel 做 pivot table，現在一句話就搞定。」

## Phase 2.5：AI 能與不能（3 min）

> 在進入投票之前，花 3 分鐘幫大家建立正確期待。

**講稿**：

> 「剛才大家看到 AI 可以寫網頁、分析資料，看起來很厲害。
> 但在我們往下走之前，先花一分鐘聊聊它的邊界，這樣等等實作時你們會更知道怎麼用。」

**AI 擅長的事（用剛才的 demo 舉例）**：
- 讀懂資料、做計算、找規律（→ 剛才的客戶分析）
- 寫文字：報告、Email、翻譯（→ 等等會看到）
- 記住這次對話的上下文（→ 剛才我們連續追問，不用重新描述）
- 把重複的邏輯自動化（→ 等等深入主題會做）

**AI 做不到的事**：
- 即時資訊可以上網搜尋，但搜尋結果不保證 100% 正確（還是要自己驗證）
- 不會登入你的後台系統幫你操作（今天做到「幫你整理好設定」，後台自動化是下一階段）
- 不保證 100% 正確（數字、日期一定要自己 double check）

**一句話總結**：

> 「把 AI 當成一個反應超快、但需要你 double check 的實習生。你給方向，它幫你跑腿。」

→ 接著進入投票：「OK，那我們來選今天要深入做哪兩個主題」

## Phase 3：深入實作 Demo 劇本

### 主題 A：Coupon 設定自動化（45 min）

**釐清需求（10 min）**

問學員：
1. 「你們現在是怎麼設定 coupon 的？一張一張手動輸入？」
2. 「設定一批 coupon 大概要花多久？」
3. 「最容易出錯的地方是什麼？」

**即時示範（25 min）**

Step 1 — 讀取並驗證 coupon 規則表：

（📎 上傳 `data/coupon_rules.csv`）

```
我上傳了一份 coupon 規則表 CSV，先幫我做資料驗證：
1. 有沒有日期格式錯誤的？
2. 有沒有折扣值不合理的（例如百分比超過 100）？
3. 有沒有重複的 coupon code？
4. 列出所有需要人工確認的項目
```

Step 2 — 產生結構化的設定檔：
```
根據剛才驗證通過的 coupon 規則，幫我產生：
1. 一份格式化的設定清單（每張 coupon 一個區塊）
2. 包含所有欄位的檢查清單，方便行銷人員逐一確認
3. 輸出成可以貼到 Google Sheets 的格式
```

Step 3 — 產生批次操作摘要：
```
根據這些 coupon 規則，幫我產一份行銷主管可以看的摘要報告：
- 按月份列出：每個月有幾個活動在跑
- 哪些 coupon 可以疊加使用
- 預估的折扣成本（假設每張 coupon 都用到 max_uses 的 50%）
- 用中文，格式要清楚好讀
```

**成果展示（10 min）**
- 展示驗證 → 設定檔 → 摘要報告的完整流程
- 問學員：「如果每次設定新 coupon 都先跑這套流程，會幫到你們嗎？」

### 主題 B：Tracking Code（UTM）生成（45 min）

**釐清需求（10 min）**

問學員：
1. 「你們現在 UTM 是怎麼產的？手動拼？有模板嗎？」
2. 「一個活動大概要產幾組 UTM？」
3. 「最怕的是什麼？拼錯？格式不一致？」

**即時示範（25 min）**

Step 1 — 從活動表產 UTM：

（📎 上傳 `data/campaign_tracking.csv`）

```
我上傳了一份行銷活動追蹤表 CSV，請為每一筆活動自動產生 UTM 參數：

規則：
- utm_source = channel 欄位值
- utm_medium = medium 欄位值
- utm_campaign = 活動名稱轉成 lowercase + 底線（中文轉拼音或保留英文）
- utm_content = target_market
- utm_term = 留空

輸出完整的 URL（landing_page + UTM 參數），格式化成表格。
```

Step 2 — 輸出成可直接使用的格式：
```
把剛才的 UTM URL 清單：
1. 輸出成 CSV 格式（可以直接匯入 Google Sheets）
2. 每個 URL 旁邊加上活動名、管道、負責人
3. 另外產一份「URL 縮短對照表」的模板
```

Step 3 — 整理成可操作的投放清單：
```
把剛才的 UTM URL 清單整理成一份 Google Sheets 格式：
- 每列包含：活動名稱、管道、負責人、完整 UTM URL、短網址預留欄位
- 另外幫我產一份「投放 checklist」，列出每個管道上線前要確認的事項
- 格式要可以直接複製貼到 Google Sheets
```

**成果展示（10 min）**
- 展示 CSV → UTM URL → 投放清單的完整流程
- 問：「下次新活動，你只要更新那張 CSV，重跑一次就好了」

### 主題 C：名單分析（45 min）

**釐清需求（10 min）**

問學員：
1. 「你們拿到名單後第一件事是做什麼？」
2. 「通常會怎麼分群？依消費金額？依國家？」
3. 「分完群之後會做什麼？寄 EDM？做活動？」

**即時示範（25 min）**

Step 1 — 自動分群：

（📎 上傳 `data/customer_list.csv`）

```
我上傳了一份客戶名單 CSV，幫我做客戶分群：

分群規則：
- VIP 流失風險：Gold/Platinum 但超過 60 天沒消費
- 高潛力新客：Bronze 但平均消費 > 150 USD
- 忠實回購客：訂單數 > 15 且近 30 天有消費
- 沉睡客戶：超過 90 天沒消費

列出每群的人數、代表客戶、平均消費金額。
```

Step 2 — 產出行銷建議：

（📎 追加上傳 `data/coupon_rules.csv`）

```
根據剛才的分群結果，幫我產出每群的行銷策略建議：
1. 建議的溝通管道（EDM / LINE / FB 廣告 etc.）
2. 建議的 coupon 類型（從我剛上傳的 coupon_rules.csv 中挑選適合的）
3. 建議的訊息方向（一句話描述）
4. 預估觸及人數和轉換率
```

Step 3 — 產出 re-engagement email：
```
針對「VIP 流失風險」那群客戶，幫我寫一封 re-engagement email：
- 語氣：溫暖、不要有壓力
- 提到他們之前喜歡的旅遊類型
- 附上一個專屬回歸優惠
- 產出中文版和英文版
```

**成果展示（10 min）**
- 展示分群 → 策略 → Email 的完整鏈路
- 問：「從名單到可以發出去的 email，剛才花了多少時間？」

### 主題 D：報表管理（45 min）

**釐清需求（10 min）**

問學員：
1. 「你們每個月的報表是怎麼整理的？」
2. 「資料來源有幾個？要手動合併嗎？」
3. 「老闆最想看的數字是什麼？」

**即時示範（25 min）**

Step 1 — 合併多月報表：

（📎 同時上傳 `data/monthly_report_2026_01.csv`、`monthly_report_2026_02.csv`、`monthly_report_2026_03.csv`）

```
我上傳了三份月報 CSV（1 月、2 月、3 月），幫我：
1. 合併成一份完整的 Q1 報表
2. 計算每個管道的 Q1 總計（impressions, clicks, conversions, spend, revenue）
3. 計算每個管道的季度 ROAS
4. 用表格呈現
```

Step 2 — 趨勢分析：
```
根據合併後的 Q1 資料，幫我分析：
1. 哪些管道的 ROAS 在上升？哪些在下降？
2. 哪個市場成長最快？（看 revenue 成長率）
3. 有沒有異常值？（某個數字突然跳很多或掉很多）
4. 用重點摘要的方式呈現，適合給主管看
```

Step 3 — 產出主管報告：
```
根據以上分析，幫我寫一份 Q1 行銷成效摘要：
- 開頭：3 個 key highlights（最好的消息）
- 中間：各管道表現比較
- 結尾：Q2 建議方向
- 風格：簡潔、數據驅動、適合在週會報告
- 中文
```

**成果展示（10 min）**
- 展示三份 CSV → 合併 → 趨勢分析 → 主管報告
- 問：「你們現在每月整理這些大概花多久？」

## Phase 5：收尾 — Claude vs Gemini 怎麼選 + 搭配建議（10 min）

> 詳細比較表見 [ai-tools-comparison.md](ai-tools-comparison.md)，以下是現場講的精華版。

**開場**：

> 「今天我們主要用 Claude Desktop，但公司也有 Gemini。最後幾分鐘幫大家搞清楚：
> 什麼時候開 Claude，什麼時候開 Gemini。」

**秀這張表**（投影）：

| 我想要... | 用這個 | 一句話原因 |
|---|---|---|
| 分析 CSV/Excel | **Claude Desktop** | 上傳 → 問問題 → 拿圖表和結論 |
| 寫報告 / Email / 文案 | **Claude Desktop** | 結構化寫作最穩，繁中最自然 |
| 辨識截圖、照片、掃描文件 | **Gemini** | 圖像辨識最準（多模態最強） |
| 分析 Google Sheets | **Gemini** | 直連 Google Drive，不用下載上傳 |
| 需要生成配圖 | **Gemini** | Imagen 內建，描述就生圖 |

**一句話記住**：

> 「文字分析找 Claude，看圖生圖找 Gemini。」

**搭配流程**（口述 + 畫在白板上）：

> 「實際工作上，一件事可能兩個都用到：」
>
> 1. 收到截圖/照片 → **Gemini** 辨識整理成文字
> 2. 拿到文字資料 → **Claude Desktop** 深入分析、產報告
> 3. 報告需要配圖 → **Gemini** 生成
>
> 「兩個工具各有擅長，搭配著用效果最好。」

**補充（如果有人問「可以連其他工具嗎」）**：

> 「Claude 有一個叫 MCP 的 Connector 機制，可以連 Slack、Google Calendar、Jira 等服務。
> 設定上需要一些額外步驟，但你可以直接問 AI 怎麼設定，它會一步步教你。」

**收尾金句**：

> 「今天學的 prompt 技巧——指定角色、分步驟、給範例——在 Claude 和 Gemini 上都通用。
> 帶走的是方法，不是只有一個工具。」
